토큰화 주요 설정

  • MAX_INPUT_LENGTH (입력)
    2048 토큰
    EDA 기반 및 모델 한계 고려
  • MAX_TARGET_LENGTH (출력)
    512 토큰
    요약 태스크를 커버하기 위해 설정

토큰화 함수 (preprocess_function)

def preprocess_function(examples):
    # 1. Input (Source) 토큰화
    model_inputs = tokenizer(
        examples['input_text'], 
        max_length=MAX_INPUT_LENGTH, 
        truncation=True
    ) 
    
    # 2. Target (Label) 토큰화
    with tokenizer.as_target_tokenizer(): 
        labels = tokenizer(
            examples['target_text'], 
            max_length=MAX_TARGET_LENGTH, 
            truncation=True
        ) 

    # 3. 모델이 사용할 'labels' 필드에 할당
    model_inputs['labels'] = labels['input_ids'] 
    return model_inputs

토큰화는 `num_proc=4` 설정으로 병렬 처리되어 빠르게 완료되었습니다 (Train: 55초, Val: 6초).

⚙️ 8단계: 학습 설정 (TrainingArguments)

핵심: A100 GPU 환경에서 안정적인 학습을 위한 메모리 관리 및 성능 최적화 설정이 적용되었습니다.

메모리/OOM 안정화

  • per_device_train_batch_size: 8
  • gradient_accumulation_steps: 2
  • PYTORCH_CUDA_ALLOC_CONF: "expandable_segments:True"
  • = 유효 배치 크기 16 유지, 메모리 부담 감소

학습 안정화

  • max_grad_norm: 1.0 (그래디언트 클리핑)
  • learning_rate: 3e-5
  • warmup_ratio: 0.05 (초반 안정화)
  • num_train_epochs: 1

A100 성능 최적화

  • bf16: True (Mixed Precision)
  • TF32: True (TensorFloat32)
  • eval_steps: 1000 (잦은 검증)
  • save_steps: 1000 (잦은 저장)
  • load_best_model_at_end: True
